MCP Testing Suite · GSoC 2025 Proposal · @souvikDevloper

Build, run & debug MCP interactions — including visual MCP Apps

MCP isn't just JSON-RPC anymore. Servers now deliver rich HTML UI components to AI hosts. This project tests the full stack: transport, protocol, tool execution, and the new MCP Apps visual layer.
😖 Today's developer pain
🔴Craft JSON-RPC payloads by hand in curl — no tooling
🔴Transport errors vs protocol errors look identical
🔴No replay — re-run everything from scratch after each fix
🔴MCP Apps: ui/initialize handshake? Test it how?
🔴Host context CSS injection — no sandbox to validate themes
→
✅ After this feature
🟢Configure MCP targets like API Dash request collections
🟢Trace viewer classifies: transport / protocol / tool / ui-handshake
🟢One-click scenario replay + diff against known-good baseline
🟢MCP Apps: test ui/initialize handshake with live message log
🟢Host context sandbox — inject light/dark themes, validate render
